FORM STRUCTURE

All the form controls live in the < form > element. It should always carry the action attribute and usually will have a method and id attribute as well. Method : Form can be sent using one of two method :

  • GET : the values from the form are added at the end of the URL (specified in the action attribute). use this for :
    • Short form (search boxes for instance)
    • When you are just retrieveing data with no need to store it or delete it from a database
  • POST : The values are sent to HTTP headers. You should use the Post method if :
    • user can upload a file
    • you need to add or delete info in a database
    • your form is gigalong
    • sensitive information is required (Password)

We also give as rule of thumb an id to forms, in order to enable scripts to do their job. (Check if a required field has a value for instance)

  • maxlength is quite common for database where there is a size lkimit for example.

RADIO BUTTON

The input type radio is used when a choice has to be made between few options type=radio

  • name should be the same for all the radio buttons as it is sent to the server along with the value they chose.
  • value is what is sent to the server depending on which option is used
  • checked indicates what should be selected "by default" when the page is opened

Careful, never use radio buttons is the option should be unselected or empty by the user.

EXAMPLE

What position do you play ?
Support Adc Mid Jungle Top

CHECKBOX

< input type="checkbox" > is used when several options can be selected and unselected.

  • name should be the same for all options, as always, for the same reasons
  • value is what is sent to the server depending on the option chosen
  • checked indicates what should be selected by default, if needed.

EXAMPLE

LABELLING FORM CONTROLS

Each form control should have a label assigned, it helps with accessibility (for people with visual impairment for instance)
There is two ways to apply a label to a frm control :

  • By wrapping the form control code around opening and closing label tags
  • Give an id to the form control and use the for attribute to label it example : < label for="Male" > Male < /label >

Common rules :

  • labels are usually placed to the left for :
    • Text inputs
    • text areas
    • drop lists
    • select boxes
    • File upload
  • labels are usually placed to the right for :
    • Individual checkboxes
    • Individual radio buttons
